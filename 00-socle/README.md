# 00-socle

Le socle définit le langage commun du dépôt. Il fixe les objets, identifiants, taxonomies et règles de passage entre les modules.

## Contenu

```text
schemas/              définitions des objets structurants
taxonomies/           listes contrôlées : risques, registres, niveaux de preuve
conventions/          nommage, identifiants, formats de fichiers
prompts-communs/      règles transversales réutilisables
```

## Règle

Aucun module ne doit inventer seul ses objets ou ses catégories. Toute donnée destinée à circuler entre modules doit s’appuyer sur le socle.
